WHAT IS ACA?
--------------------------------------------------------------

ACA (Agentic Cloud Assistant) is an AI-powered AWS
infrastructure management tool built on the Model Context
Protocol (MCP). Instead of clicking through the AWS console,
you talk to ACA — it understands your cloud, finds the
problems, writes the fixes, and waits for your approval before
touching anything.

It works in three places simultaneously:
  • Your browser (React web app)
  • Claude Desktop (via MCP stdio transport)
  • Claude Code CLI (via MCP HTTP transport)

--------------------------------------------------------------
KEY FEATURES
--------------------------------------------------------------

1. AWS Infrastructure Scanning
   - Scans EC2, S3, IAM, Security Groups, and VPCs in one click
   - Maps every Security Group to its attached resources
     (EC2, RDS, ALB, Lambda, ECS, ElastiCache) via ENIs
   - Identifies unused, orphaned Security Groups
   - Auto-refreshes every 90 seconds after first scan

2. Security Analysis
   - 7-rule security engine covering the most critical misconfigs:
     open SSH (port 22) and RDP (port 3389) to the internet,
     public S3 buckets, missing MFA, root account usage,
     over-permissive IAM policies, and more
   - Severity-ranked findings: HIGH / MEDIUM / LOW
   - 0–100 security health score shown in the top bar
   - One-click direct fix: removes dangerous ingress rules
     instantly without writing Terraform
   - LLM-generated plain-English summary of every finding

3. Cost Monitoring
   - Current month spend, 3-month trend, per-service breakdown
   - Anomaly detection: flags any service with a 20%+ spike
   - LLM optimisation recommendations based on live cost data

4. Terraform HCL Generation
   - Describe any AWS resource in plain English
   - LLM scans your existing infra first (real VPC/SG/subnet IDs)
     to generate HCL that won't conflict with live resources
   - Auto-validates syntax via `terraform validate`

5. Human-in-the-Loop Execution
   - `terraform plan` runs first — you see the full diff
   - `terraform apply` only fires on your explicit approval
   - `terraform destroy` available for instant rollback
   - EC2 instances with new key pairs generate a downloadable
     .pem file right in the browser

6. Agentic Security Remediation
   - One click: the autonomous agent scans your AWS, picks the
     highest-severity finding, writes a Terraform fix, plans it,
     and presents a plain-English summary for your sign-off
   - You approve or reject — the agent never applies without you

7. Infrastructure Chat
   - Ask questions about your live AWS environment in plain English
   - Every message is grounded in a fresh 9-source AWS scan
     (EC2, S3, IAM, SGs, VPCs, SG usage, cost data + more)
   - Powered by Groq, Anthropic, or fully local Ollama

8. RAG Knowledge Base
   - Upload your own PDF or text security documentation
   - Ask questions and get answers grounded in your docs
   - Backed by ChromaDB vector store + SentenceTransformer
     embeddings — all local, no external embedding API

9. Execution History & Audit Log
   - Every plan, apply, destroy, and rollback is logged
   - Mark findings as resolved without deleting them (audit-safe)
   - Full terminal output stored per execution

10. Multi-Client MCP Support
    - Same backend serves the browser UI, Claude Desktop,
      and Claude Code CLI simultaneously via MCP protocol
    - 30+ tools available as slash commands in Claude Code

HOW IT WORKS (3-STEP FLOW)
--------------------------------------------------------------

Step 1 — Scan
  Enter your AWS credentials and click Scan. ACA runs five
  parallel scanners across EC2, S3, IAM, Security Groups,
  and VPCs and populates the dashboard in seconds.

Step 2 — Analyse
  Click Analyse on the Security or Cost panel. The 7-rule
  engine flags every misconfiguration, assigns severity, and
  an LLM writes a plain-English summary of what's wrong
  and why it matters.

Step 3 — Fix & Approve
  Choose a fix path: direct revoke (instant, no Terraform),
  manual Terraform generation, or the autonomous agent.
  ACA shows you the full plan before anything changes.
  You approve. It applies.

SECURITY NOTE
--------------------------------------------------------------

  ACA never stores your AWS credentials on disk. Keys are
  passed at request time and forwarded directly to boto3.
  The human-in-the-loop approval gate ensures no infrastructure
  change is ever applied without explicit user confirmation.
